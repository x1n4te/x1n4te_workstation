## Overview

The AI Insights recommendations screen was redesigned to show **more focused, contextual information** — limiting noise while helping parents instantly identify the most relevant advice.

---

## 1. Date Display on Every Recommendation

### What It Is
Every recommendation card now shows the exact date it was generated by the AI.

### Format
```
Apr 25, 2026
```

### Location
Bottom-right of each card, in the new footer row.

### Why It Matters
Parents can tell at a glance whether an insight is from today, yesterday, or last week — helping them decide whether the advice is still relevant to their child's current routine.

---

## 2. Three-Per-Category Limit

### What It Is
Instead of showing *all* AI recommendations at once, the screen now shows **only the 3 most recent recommendations per category** (Sleep, Meals, Education, Screen Time, General).

### How It Works
```
1. Group all recommendations by category
2. Within each category, sort by date (newest first)
3. Keep only the top 3
4. Discard older recommendations from that category
```

### Example
If the AI has generated 8 Sleep recommendations over the past month:
- **Before:** All 8 shown, creating a wall of text
- **After:** Only the 3 newest Sleep recommendations shown

### Why It Matters
- **Prevents overwhelm** — Parents aren't faced with dozens of cards
- **Focuses on recent insights** — Older advice may no longer apply as routines change
- **Improves scannability** — Screen loads faster and is easier to read

---

## 3. Recency Badges

### What They Are
Color-coded labels that tell parents how fresh each recommendation is **within its category**.

### Badge System

| Badge | Color | Meaning |
|-------|-------|---------|
| **Latest** | Green (`#10B981`) | Most recent recommendation in this category |
| **2nd Latest** | Amber (`#F59E0B`) | Second most recent in this category |
| **Oldest** | Gray (`#94A3B8`) | Third most recent (oldest of the 3 shown) |

### Visual Example

### Important Note
"Oldest" does **not** mean outdated or irrelevant — it simply means it's the third-most-recent insight in that category. All three shown insights are the freshest available.

## 4. Filter Behavior

### How Filters Interact with the 3-Limit
When a parent selects a filter (e.g., "Sleep" or "High Priority"):
1. The filter is applied first (show only Sleep recommendations)
2. Then the 3-per-category limit is applied
3. Only the 3 most recent Sleep recommendations are shown

### Why This Is Better
- Filtering to "Sleep" won't show 15 old Sleep cards — just the 3 most relevant
- The "All" view remains scannable because each category is capped

---

## Technical Summary

### Data Flow
```
Raw Recommendations (from Supabase)
    ↓
Apply Active Filter (All / Sleep / High Priority / etc.)
    ↓
Group by Category (sleep, meal, education, screen_time, general)
    ↓
Sort Each Group by Date (newest → oldest)
    ↓
Slice to Top 3 per Group
    ↓
Assign Recency Rank (0=Latest, 1=2nd Latest, 2=Oldest)
    ↓
Sort Final List by Category Name → Date
    ↓
Render Cards with Date Footer + Recency Badge
```

### Code Constants
```typescript
const RECENCY_LABELS = ['Latest', '2nd Latest', 'Oldest'];
const RECENCY_COLORS = ['#10B981', '#F59E0B', '#94A3B8'];
```

## Client FAQ

**Q: Will older recommendations be deleted from the database?**  
A: No. All recommendations are still stored in Supabase. Only the *display* is limited to 3 per category.

**Q: What if a category has fewer than 3 recommendations?**  
A: All available recommendations for that category are shown. No placeholders or empty states appear.

**Q: Do the recency badges change as new recommendations are generated?**  
A: Yes. When the AI generates a new recommendation for a category, the badges automatically recompute. The new one becomes "Latest," the previous "Latest" becomes "2nd Latest," and so on.
