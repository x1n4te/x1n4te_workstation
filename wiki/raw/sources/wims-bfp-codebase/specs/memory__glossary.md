# WIMS-BFP Domain Glossary

- **WIMS-BFP:** Web Incident Monitoring System - Bureau of Fire Protection.
- **Centralized Single-Tier Validation:** The workflow where ONLY the National Validator cross-references and verifies unverified civilian/regional reports.
- **Civilian Reporter:** Public user submitting unverified crowdsourced fire incidents via the public VPS portal.
- **Regional Encoder:** BFP personnel using the offline-first PWA to log official data or upload encrypted offline bundles.
- **National Validator:** The sole BFP authority capable of changing an incident's state from `unverified` to `verified`.
- **National Analyst:** Read-only executive role utilizing PostGIS heatmaps and statistics.
- **System Administrator:** IT personnel overseeing Keycloak, Suricata logs, and reviewing XAI narratives.
- **XAI (Explainable AI):** The localized Qwen2.5-3B model used solely to translate raw Suricata JSON alerts into human-readable forensic narratives.
- **Sovereign Trust / Sovereign Core:** The logically isolated backend environment ensuring the government retains absolute control over data and AI inference without third-party cloud reliance.
- **Offline Bundle:** A mathematically sealed `.wims` file (AES-256-GCM) generated by the PWA during offline mode, synchronized when the network returns.
