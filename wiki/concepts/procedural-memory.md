# Procedural Memory (SKILL.md)

**Framework:** Hermes-Agent (Nous Research)  
**Layer:** Memory Architecture — Layer 3  
**File Location:** `~/.hermes/skills/*.md`  

---

## Definition

Procedural memory is Hermes-Agent's **Layer 3** memory — self-authored, task-specific methodology documents written in SKILL.md format after successful completion of complex tasks. It is the concrete output of the closed learning loop's Extract phase.

Unlike system prompts (which are hand-written and static) or prompt templates (which are still human-authored), SKILL.md files are **autonomously generated by the agent itself** after encountering and solving novel problems.

---

## SKILL.md Structure

A skill document contains:

```yaml
---
name: skill-name
description: What this skill handles
trigger: "condition that activates this skill"
requires_toolsets: [web, terminal]      # Conditional visibility
requires_tools: [search_files, terminal] # Specific tool requirements
fallback_for_tools: [...]                 # Activates when primary tools offline
---

# Approach
High-level strategy for this task type.

# Steps
1. Step one
2. Step two
```

### Key Metadata Fields

| Field | Purpose |
|---|---|
| `requires_toolsets` | Hides skill from system prompt if toolset is disabled (air-gapped env) |
| `requires_tools` | Strict tool dependency |
| `fallback_for_tools` | Auto-surfaces when primary API systems are offline |
| `trigger` | Condition string for skill activation |

---

## Conditional Visibility (Context Window Protection)

The `requires_toolsets` mechanism prevents context pollution. Example:
- A `web-search` skill has `requires_toolsets: [web]`
- If agent is running with web toolset disabled, skill is **automatically hidden** from system prompt and slash commands
- Result: context window not polluted with irrelevant capabilities

This is critical for maintaining high processing speed in constrained environments.

---

## Zero-Context-Cost Turns

By pre-compiling task methodologies into SKILL.md rather than re-deriving them via LLM at runtime, Hermes-Agent achieves **"zero-context-cost turns"** for routine complex tasks:
- Agent encounters similar problem
- Skill is retrieved and injected
- LLM skips planning phase → immediately executes
- TTFT and token expenditure drastically reduced

---

## WIMS-BFP Application

WIMS-BFP audit/remediation patterns (RLS policies, Celery task fixes, Suricata ingestion pipelines) are ideal candidates for SKILL.md storage. After manually guiding the agent through an RLS fix once, the closed learning loop should compile it into a skill for future zero-shot execution.
