# skill.md — Open Standard for Agent Skills

**Source:** Mintlify blog (January 21, 2026)
**Spec:** https://agentskills.io
**Maintained by:** Anthropic + open community
**RFC:** Cloudflare skills RFC, Vercel skills CLI

---

## What Is skill.md?

A markdown file that lives alongside documentation, describing how best agents should use a product. Skill files can be installed into 20+ major coding agents including Claude Code, OpenCode, and Cursor via Vercel's `skills` CLI.

All Mintlify documentation sites now contain a `/.well-known/skills/default/skill.md` file at their base URL.

---

## Why Skills Matter

Agents have access to all your documentation, yet they often write horrible code.

This isn't because models are unintelligent — with perfect context they excel. It's because:

1. **Documentation is written for humans** — humans browse incrementally, agents can't hold entire docs in context
2. **Features are scattered** — best practices spread across dozens of pages
3. **Models are out of date** — trained on past data, need up-to-date context
4. **Token limits** — can't hold massive amounts of tokens and still perform at full capacity

**skill.md answers:** What information should agents ALWAYS have in context? How do you keep it up to date?

With the right up-to-date context, code quality produced by agents is massively improved.

---

## What Belongs in skill.md

Skill.md is the information you want agents to ALWAYS have when working with your product:

- **Personal taste** — your coding style preferences
- **Capabilities** — what the tool can do
- **Limitations** — what it can't do
- **Best practices** — how to use it correctly
- **Gotchas** — common mistakes agents make repeatedly

Documentation usually scatters this across dozens of pages, or skips it entirely because it would overwhelm human readers. Skill.md consolidates it for agents.

---

## How to Design skill.md (Mintlify's Approach)

### 1. Decision Tables for Tribal Knowledge

Instead of describing every component, use tables:

```
Need                        Use
Hide optional details       <Accordion>
Sequential instructions     <Steps>
Code in multiple languages  <CodeGroup>
```

Agents struggle making intelligent choices. If you can confidently make choices for them, you reduce their error rate.

### 2. Explicit Boundaries

Separate "what agents can configure" from "requires dashboard setup." This prevents agents from attempting impossible tasks.

### 3. Gotchas Section

Include mistakes agents make repeatedly without explicit guidance. Example: "Never use mint.json — it's deprecated, only use docs.json."

### 4. Link Out for More Information

skill.md is a cheat sheet, NOT a replacement for docs. Point to full documentation via llms.txt for anything needing more context.

### 5. Concise Style

See Vercel web design guidelines — written in a concise style not ideal for human browsing, but excellent for agents.

## How It Works With Mintlify

- skill.md lives at `/.well-known/skills/default/skill.md`
- Also served at `/skill.md` for convenience
- Mintlify regenerates the skill.md file every time docs are updated
- Always contains the latest information
- Custom skill.md can replace the autogenerated one by placing it in the repo root

---

## Discovery & Installation

```bash
# Automatic discovery — just provide docs URL
npx skills add https://mintlify.com/docs

# Point directly to skill file
npx skills add https://example.com/.well-known/skills/default/skill.md
```

Supported agents: Claude Code, OpenCode, Cursor, and 20+ others.

## Connection to Hermes Skills

Hermes uses SKILL.md files (same concept, slightly different format):

| Aspect | Agentskills Standard | Hermes SKILL.md |
|--------|---------------------|-----------------|
| Format | Markdown | YAML frontmatter + Markdown |
| Location | `/.well-known/skills/` | `~/.hermes/skills/` |
| Discovery | `npx skills add` | `hermes skills install` |
| Loading | Agent loads on task match | Agent loads on trigger or manual `/skill` |
| Auto-update | Regenerated with docs | Manually updated |
| Content | Decision tables, gotchas, boundaries | Same — procedures, pitfalls, verification |

Both solve the same problem: **give agents the right context at the right time.**
