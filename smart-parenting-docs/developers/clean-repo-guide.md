# Clean Repository Guide — First Run on Windows

This guide ensures your repository contains **only the files needed** for a fresh first run on Windows. No build artifacts, no secrets, no platform-specific junk.

---

## Philosophy

A clean repository should allow a new developer to:

1. `git clone` the repo
2. Run `npm install`
3. Create their own `.env` from `.env.example`
4. Run `npx expo start`

Nothing more. Everything else is generated.

## What Gets Deleted (Generated/Local)

These should **never** be in version control. Delete them before sharing the repo.

| Path | Why |
|------|-----|
| `node_modules/` | Installed by `npm install`. Platform-specific binaries inside. |
| `.expo/` | Generated by Expo CLI. Cache, settings, and local server state. |
| `.expo-shared/` | Legacy Expo sharing cache. |
| `android/` | Generated by `npx expo prebuild`. Native Android project. |
| `ios/` | Generated by `npx expo prebuild`. Native iOS project. |
| `.env` | Contains real secrets. Each developer creates their own. |
| `*.log` | Debug logs. |
| `npm-debug.log*` | npm crash logs. |
| `yarn-error.log*` | Yarn crash logs. |
| `.DS_Store` | macOS Finder metadata. |
| `Thumbs.db` | Windows Explorer thumbnail cache. |
| `.vscode/` | Editor settings (personal preference). |
| `.idea/` | JetBrains IDE settings. |
| `coverage/` | Test coverage reports (generated by `npm test`). |
| `*.jks` | Android keystores (secrets). |
| `*.p8` | Apple Push Notification keys (secrets). |
| `*.p12` | Certificates (secrets). |
| `*.key` | Private keys (secrets). |
| `*.mobileprovision` | iOS provisioning profiles. |
| `dist/` | Build output directory. |
| `web-build/` | Expo web build output. |
| `scripts/` | Dev helper scripts (optional — see note below) |

### Note on `scripts/`

The project originally had helper scripts like `create_test_user.js` and `remove-dom-webview.js`. These are **optional** for first run:

- **Keep** if they are genuinely needed for setup.
- **Delete** if they are one-off fixes or dev conveniences.

For a clean repo, move them to a `tools/` directory with a note, or delete them and document the commands inline.

## Step-by-Step Cleanup on Windows

Open **PowerShell** in your project root and run these commands one by one.

### 1. Delete Generated Directories

```powershell
# Node modules (reinstallable)
if (Test-Path node_modules) { Remove-Item -Recurse -Force node_modules }

# Expo cache & generated files
if (Test-Path .expo) { Remove-Item -Recurse -Force .expo }
if (Test-Path .expo-shared) { Remove-Item -Recurse -Force .expo-shared }

# Prebuild native directories (if they exist)
if (Test-Path android) { Remove-Item -Recurse -Force android }
if (Test-Path ios) { Remove-Item -Recurse -Force ios }

# Build outputs
if (Test-Path dist) { Remove-Item -Recurse -Force dist }
if (Test-Path web-build) { Remove-Item -Recurse -Force web-build }

# Coverage reports
if (Test-Path coverage) { Remove-Item -Recurse -Force coverage }
```

### 2. Delete Log & Secret Files

```powershell
# Log files
Remove-Item -Force *.log -ErrorAction SilentlyContinue
Remove-Item -Force npm-debug.log* -ErrorAction SilentlyContinue
Remove-Item -Force yarn-error.log* -ErrorAction SilentlyContinue
Remove-Item -Force yarn-debug.log* -ErrorAction SilentlyContinue

# Real .env file (keep .env.example!)
if (Test-Path .env) { Remove-Item -Force .env }

# OS junk
Remove-Item -Force .DS_Store -ErrorAction SilentlyContinue
Remove-Item -Force Thumbs.db -ErrorAction SilentlyContinue

# Editor settings (optional — only if you want to force clean)
# if (Test-Path .vscode) { Remove-Item -Recurse -Force .vscode }
# if (Test-Path .idea) { Remove-Item -Recurse -Force .idea }
```

### 3. Clean Git Index (If Already Tracked)

If you previously committed files that should be ignored:

```powershell
# Remove everything from the git index (staging area)
git rm -r --cached .

# Re-add only what should be tracked (respects .gitignore)
git add .

# Check what changed
git status
```

You should see:
- **Deleted:** `node_modules`, `.expo`, `.env`, etc.
- **Untracked:** nothing (if .gitignore is correct)
- **Modified:** nothing unexpected

### 4. Commit the Clean State

```powershell
git commit -m "chore: clean repository for first-run setup

- Remove node_modules, .expo, android, ios, build outputs
- Remove .env (secrets should never be committed)
- Remove log files and OS metadata
- Ensure .gitignore covers all generated files"
```

## The `.env.example` File

Since `.env` is ignored, you **must** provide a template so new developers know what to create.

Create `.env.example` (committed to git):

```env
# Supabase Configuration
# Get these from your Supabase project: Project Settings → API
EXPO_PUBLIC_SUPABASE_URL=https://your-project-url.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here

# Optional: OpenRouter API Key (only needed for local Edge Function testing)
# OPENROUTER_API_KEY=your-openrouter-key-here
```

**Instructions for first-time developers:**
1. Copy `.env.example` to `.env`
2. Replace the placeholder values with your actual Supabase credentials
3. Never commit `.env`

## Verification Script

Run this PowerShell script to verify your repository is clean before pushing.

Save as `verify-clean.ps1` in the project root (add it to `.gitignore` after use, or delete it after running):

```powershell
# verify-clean.ps1
# Run this to check if your repo is clean for first-run

$errors = 0

$forbiddenPaths = @(
    "node_modules",
    ".expo",
    ".expo-shared",
    "android",
    "ios",
    "dist",
    "web-build",
    "coverage",
    ".env",
    "*.log"
)

Write-Host "Checking for forbidden files/directories..." -ForegroundColor Cyan

foreach ($path in $forbiddenPaths) {
    $matches = Get-ChildItem -Path $path -ErrorAction SilentlyContinue
    if ($matches) {
        Write-Host "  ❌ FOUND: $path" -ForegroundColor Red
        $errors++
    }
}

# Check .gitignore exists and has key entries
if (-not (Test-Path .gitignore)) {
    Write-Host "  ❌ MISSING: .gitignore" -ForegroundColor Red
    $errors++
} else {
    $gitignore = Get-Content .gitignore -Raw
    $required = @("node_modules/", ".env", ".expo/", "android/", "ios/")
    foreach ($entry in $required) {
        if ($gitignore -notmatch [regex]::Escape($entry)) {
            Write-Host "  ⚠️  .gitignore missing: $entry" -ForegroundColor Yellow
            $errors++
        }
    }
}

# Check .env.example exists
if (-not (Test-Path .env.example)) {
    Write-Host "  ❌ MISSING: .env.example (template for secrets)" -ForegroundColor Red
    $errors++
}

# Check package.json exists
if (-not (Test-Path package.json)) {
    Write-Host "  ❌ MISSING: package.json" -ForegroundColor Red
    $errors++
}

if ($errors -eq 0) {
    Write-Host "`n✅ Repository is clean and ready for first run!" -ForegroundColor Green
} else {
    Write-Host "`n❌ Found $errors issue(s). Fix before sharing the repository." -ForegroundColor Red
    exit 1
}
```

Run it:

```powershell
.\verify-clean.ps1
```

## Post-Cleanup First Run

After cleaning the repository, the first run on a new Windows machine is:

```powershell
# 1. Clone
git clone https://github.com/YOUR_USERNAME/smart-parenting-app.git
cd smart-parenting-app

# 2. Install dependencies
npm install

# 3. Create environment file
copy .env.example .env
# Edit .env with your Supabase credentials

# 4. Start the app
npx expo start
```

That's it. No cleanup, no fixing tracked junk, no conflicts.

## Quick Checklist Before Pushing to Git

- [ ] `node_modules/` is gone
- [ ] `.expo/` is gone
- [ ] `android/` and `ios/` are gone (unless you intentionally ejected)
- [ ] `.env` is gone (`.env.example` exists instead)
- [ ] No `*.log` files
- [ ] No `.DS_Store` or `Thumbs.db`
- [ ] `.gitignore` covers all generated files
- [ ] `package-lock.json` is present (for reproducible installs)
- [ ] `verify-clean.ps1` passes with zero errors
